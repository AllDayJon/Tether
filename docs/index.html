The idea

Stop re-explaining what you just did.

You're the one doing the work. You're in the terminal — SSH'd into a box, tailing logs, running commands, figuring things out. When you want a second opinion from AI, you have to stop, copy output, explain what you've been doing, and rebuild the context from scratch. Every single time.

Tether fixes that. It watches your terminal in the background while you work. When you ask a question, Claude already knows what's been happening. No copy-paste. No re-explaining. No context loss.

You work normally

SSH into servers, read logs, run commands. Tether is invisible until you need it. No interruptions, no approval dialogs breaking your flow.

Tether watches silently

Your tmux panes are polled every 750ms. New output is buffered in memory, scored for relevance, and compressed — never written to disk.

Ask when you're ready

Hit your keybind. Claude already knows what's going on — it's been watching. No copy-paste, no re-explaining, no context loss.

Smart context

Everything captured. Only what matters sent.

Tether keeps a full buffer of your terminal output locally. When you ask a question, it scores every line for relevance and only sends the parts that actually matter — so Claude has real awareness without you burning through your token budget on irrelevant output.

Features

Built for real terminal work

Persistent context

Claude sees your terminal history before you ask anything. No copy-paste, no re-explaining what you've been doing.

Delta-only updates

Only new output is sent each message, not the full buffer. You're not paying for 500 lines of log output every time you ask a question.

Relevance filtering

Context is scored against your question. Irrelevant lines are dropped. Reduces per-pane context from ~200 lines to ~25 focused lines.

SSH-transparent

Commands are injected via tmux send-keys. No Tether install needed on remote hosts — works anywhere you can SSH.

Kill switch

Ctrl+K sends Ctrl+C to the work pane instantly. Stop anything, anytime, without leaving the chat window.

Conversation memory

Chat history persists across sessions and compacts automatically. Pick up right where you left off, without token bloat.

Rolling summary

A background summarizer generates a narrative of your session every 5 minutes. Claude gets the big picture without raw line-by-line history.

Token visibility

Run tether tokens to see context buffer size. Use --debug to log exactly what gets sent to Claude on each message.

Security model

Claude's output is never trusted directly

Every command Claude suggests passes through a classifier before anything happens. Claude can influence what is proposed, but never what is executed.

Hard deny
Fork bombs, pipe-to-shell (curl | bash, wget | sh). Always blocked. Not overridable by anyone.
Hard protect
sudo, output redirects (>, >>, tee), command chaining (&&, ;). Always proposed, never auto-executed.
Config rules
Your deny, protect, and allow lists in ~/.tether/config.json — full control over what is trusted.
Session allow list
Approve a command once with [a] in the proposal UI. It auto-executes for the rest of the session without touching your config.

Installation

Up in four commands

Tether is built for developers and engineers who live in the terminal. You'll need tmux for session capture, Go 1.22+ to build from source, and a Claude Code CLI subscription (claude on PATH and logged in) — Tether uses it for all AI calls, so there's no separate API key or billing.

1
Clone and install
shell 
git clone https://github.com/AllDayJon/Tether
cd tether
go install .
2
Start a tmux session and the daemon
shell 
tmux new -s work
tether start
3
Watch a pane and open chat
shell 
tether watch       # watch current pane
tether chat        # open chat split (40% right)
4
Install a keybind (optional but recommended)
shell 
tether keybind install --persist
# prefix+g now opens the chat from anywhere
